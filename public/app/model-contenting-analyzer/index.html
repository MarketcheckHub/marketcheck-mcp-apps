What This App Does

OEM product planning dashboard that quantifies which trims and content packages are actually moving for a specific model. Pulls active inventory facet counts (trim, body_type, fuel_type) from the live MarketCheck dataset, compares them to recent sold mix and Enterprise Sold Vehicle Summary rankings, and surfaces every trim's supply-vs-demand imbalance in basis points. Then samples the slowest- and fastest-moving listings, decodes their VINs, and lays out a side-by-side feature comparison so you can see whether AWD, sunroof, leather, or wheel size is the content actually driving turn rate. Outputs a 0–100 Contenting Score, oversupplied/undersupplied badges per trim, a price-by-DOM scatter, a body-type demand table, and plain-English insights ranking the largest trim mismatches and content drivers.

Who Can Use This

OEM Product Planners
Decide which trims to over-build next quarter. The supply vs demand mismatch table flags trims where inventory share is materially out of step with sold share — that's exactly where allocation should shift.
OEM Pricing & Incentives
Find oversupplied trims that are sitting on lots (high DOM, oversupplied badge) and target them with incentives or MSRP adjustments instead of brand-wide spend.
Regional Sales Managers
Pass a state code to see whether a regional market wants different content than the national mix — useful for redirecting inventory between zones.
Dealer Group Buyers
Use the price-by-DOM scatter and feature comparison grid to source inventory in the trim-and-content combinations that turn fastest in your market.
Auto Analysts / Press
Quote concrete numbers — 'X trim is undersupplied at Y bps' — backed by real listing-level data instead of OEM-reported wholesale figures.

API Endpoints Used — Full Details

Each API call explained with parameters, return values, and documentation links:

GET /v2/search/car/active

Search Active Listings

Search currently active car listings across 95%+ of US dealer inventory. Supports filters by make, model, year, ZIP, radius, price range, body type, and more. Returns listing details with photos, dealer info, and computed statistics.

ParameterTypeRequiredDescription
makestringNoVehicle make (e.g., Toyota)
modelstringNoVehicle model (e.g., Camry)
yearstringNoYear or range (e.g., 2020-2022)
zipstringNoCenter ZIP for radius search
radiusnumberNoSearch radius in miles
rowsnumberNoNumber of results (max 50)
statsstringNoAggregate stats: price,miles,dom
sort_bystringNoSort field: price, miles, dom
sort_orderstringNoasc or desc
dealer_idstringNoFilter by dealer ID
facetsstringNoFacet fields: make,model,body_type
Returns: num_found, listings[] with id, vin, price, miles, dom, dealer{}, media{}, stats{}

GET /api/v1/sold-vehicles/summary Enterprise API

Sold Vehicle Summary (Market Intelligence)

Aggregated market intelligence from sold vehicle data. Rankings by make, model, body type, state. Returns sold counts, average prices, days-on-market, and market share data.

ParameterTypeRequiredDescription
ranking_dimensionsstringNoDimensions: make, model, body_type, state, fuel_type_category
ranking_measurestringNoMeasures: sold_count, average_sale_price, average_days_on_market
ranking_orderstringNodesc or asc
top_nnumberNoTop N results
statestringNo2-letter state code
inventory_typestringNoUsed or New
Returns: rankings[] with dimension values, sold_count, average_sale_price, market_share

Build It Yourself

1. Get a Free API Key

Sign up at developers.marketcheck.com to get a free API key.

2. As an MCP App (AI Assistants)

Add the MarketCheck MCP server to Claude Desktop, VS Code, or any MCP-compatible host:

{
  "mcpServers": {
    "marketcheck-apps": {
      "url": "https://apps.marketcheck.com/mcp?api_key=YOUR_KEY"
    }
  }
}

Then ask the AI: "Use the model-contenting-analyzer tool to which trims and configs are the market buying?"

3. Embed as an iFrame

<iframe
  src="https://apps.marketcheck.com/apps/model-contenting-analyzer/dist/index.html?api_key=YOUR_KEY&embed=true"
  width="100%" height="700"
  style="border:none;border-radius:8px;"
></iframe>

4. Build from Scratch

Use the API flow diagram above as your implementation blueprint. Call the APIs in the sequence shown, handle parallel calls with Promise.all(), and render the UI components listed in "What to Render".
