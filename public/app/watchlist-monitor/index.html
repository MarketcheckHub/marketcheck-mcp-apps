What This App Does

Bloomberg-style morning signal scan for equity analysts covering the US auto sector. Monitors a 10-ticker watchlist spanning OEMs (F, GM, TM, HMC, TSLA, STLA) and dealer groups (AN, LAD, KMX, CVNA) against live MarketCheck 90-day sold and active-inventory data. Each ticker is classified ALERT / WATCH / STABLE / STRONG based on four real-time signals: volume momentum (90-day sold share vs expected), ASP spread (sold avg vs active avg — a pricing-power proxy), days-supply (active ÷ monthly-sold × 30), and discount-change (bps derived from the sold-vs-active gap). A sector summary bar rolls up industry volume trend, average ASP, EV penetration, and a macro BULLISH/BEARISH/NEUTRAL call from the signal mix. Click any ticker row to open a detail panel with a 6-month sparkline, metric breakdown, and 3–4 bullet signal analysis explaining why the flag fired. Optional `state=XX` query parameter scopes every signal to a US state.

Who Can Use This

Sell-Side Auto Analysts
Kick off the day with a 10-second scan of your auto watchlist — the table surfaces which issuers have days-supply breaching 90 days or pricing premium turning negative, so you can triage channel checks before the 9am morning call.
Buy-Side PMs / Hedge Funds
Pre-position ahead of OEM prints: the ALERT badges flag which names in your book are showing margin-stress signals (incentive escalation, ASP erosion) that consensus EPS models haven't yet reflected.
Credit Analysts
Monitor dealer-group and OEM leverage names — KMX/CVNA/AN days-supply and pricing spread are leading indicators of covenant-level dealer-channel stress 1–2 quarters before it shows up in 10-Q filings.
Automotive Equity Research Associates
Paste the ticker signals and sparklines directly into the weekly channel-check memo — each ALERT row has ready-made bullet explanations tied to real MarketCheck volume and pricing data.
Portfolio Risk Managers
Cross-ticker health-check: the macro-signal pill on the sector bar gives a one-glance read on whether the automotive book should be de-risked going into the next print cycle.

Build It Yourself

1. Get a Free API Key

Sign up at developers.marketcheck.com to get a free API key.

2. As an MCP App (AI Assistants)

Add the MarketCheck MCP server to Claude Desktop, VS Code, or any MCP-compatible host:

{
  "mcpServers": {
    "marketcheck-apps": {
      "url": "https://apps.marketcheck.com/mcp?api_key=YOUR_KEY"
    }
  }
}

Then ask the AI: "Use the watchlist-monitor tool to morning signal scan across your portfolio"

3. Embed as an iFrame

<iframe
  src="https://apps.marketcheck.com/apps/watchlist-monitor/dist/index.html?api_key=YOUR_KEY&embed=true"
  width="100%" height="700"
  style="border:none;border-radius:8px;"
></iframe>

4. Build from Scratch

Use the API flow diagram above as your implementation blueprint. Call the APIs in the sequence shown, handle parallel calls with Promise.all(), and render the UI components listed in "What to Render".
